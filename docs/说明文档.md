## 核心概念

系统采用 **RBAC**（基于角色的访问控制）模型，访问控制分三个层次：

| 层次 | 控制什么 | 配置文件 |
|------|---------|---------|
| **菜单（Menu）** | 侧边栏能显示哪些页面 | `menu-config.json` |
| **按钮权限（Permission）** | 页面上哪些按钮/操作可见 | `permission-config.json` |
| **角色（Role）** | 将菜单和权限打包成分配单元 | `role-config.json` |

```
用户 → 角色 → 菜单（页面可见性）
             → 权限（操作可见性）
```

**三层控制的区别：**

- **菜单**决定"你能去哪"——侧边栏显示哪些入口
- **权限**决定"你能干什么"——按钮/操作是否可用
- **角色**是前两者的**打包**：把若干菜单和若干权限组合成一个角色，方便批量分配
