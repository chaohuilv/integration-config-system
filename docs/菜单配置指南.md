# 菜单配置指南

> 本文档详细介绍 `menu-config.json` 的配置结构、三级菜单机制以及菜单如何控制页面可见性。

---

## 一、文件位置与结构

**路径**：`src/main/resources/config/menu-config.json`

**顶层结构**：

```json
{
  "menus": [ /* 菜单项数组 */ ]
}
```

---

## 二、字段说明

每个菜单项的字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | string | ✅ | 菜单唯一标识，与侧边栏 HTML 元素 ID 对应 |
| `name` | string | ✅ | 侧边栏显示的名称 |
| `icon` | string | ❌ | 菜单图标（emoji 或 CSS 类名） |
| `path` | string | ❌ | URL 路径片段（用于路由） |
| `pageFile` | string | ✅ | 对应的 HTML 文件路径，如 `pages/api_list.html` |
| `section` | string | ❌ | 所属分组名称，决定在侧边栏哪个折叠区域内 |
| `sortOrder` | number | ❌ | 排序序号（越小越靠前），无此字段则按配置顺序 |
| `pageType` | string | ❌ | 页面类型：`LIST` / `FORM`，不影响功能 |
| `children` | array | ❌ | 子菜单数组，支持多级嵌套 |

---

## 三、三级菜单

系统支持三级菜单嵌套，通过 `children` 字段实现：

```
一级菜单（section 分组） → 二级菜单（父级菜单） → 三级菜单（叶子节点）
```

### 示例：接口管理的三级结构

```json
{
  "code": "integration",
  "name": "集成管理",
  "section": "集成管理",
  "sortOrder": 1,
  "children": [
    {
      "code": "list",
      "name": "接口列表",
      "pageFile": "pages/api_list.html",
      "sortOrder": 1,
      "children": [
        {
          "code": "api-add",
          "name": "新建接口",
          "pageFile": "pages/api_form.html",
          "sortOrder": 1
        },
        {
          "code": "api-edit",
          "name": "编辑接口",
          "pageFile": "pages/api_form.html",
          "sortOrder": 2
        }
      ]
    }
  ]
}
```

**实际配置**中，部分菜单项是**平铺**而非嵌套的，由 `section` 字段决定分组。但上述嵌套结构同样被支持。

---

## 四、section 分组

`section` 字段将菜单归入侧边栏的折叠分组：

| section 值 | 显示位置 | 说明 |
|-----------|---------|------|
| `集成管理` | 侧边栏"集成管理"分组 | 接口配置、环境管理等 |
| `系统管理` | 侧边栏"系统管理"分组 | 用户、角色、日志、大盘、告警等 |
| `开发文档` | 侧边栏"开发文档"分组 | 接口文档、SDK文档 |

不带 `section` 的菜单项不显示在侧边栏，但通过 URL 直接访问仍然有效（配合权限控制）。

## 六、菜单与权限的关系

**重要**：`menu-config.json` 只定义菜单的**静态结构**，不包含权限绑定信息。

菜单与角色的关联由 `DataInitializer` 在启动时从 `role-config.json` 读取，并写入 `sys_role_menu` 表。

```
menu-config.json → 定义菜单结构
role-config.json → 绑定角色与菜单
sys_role_menu    → 运行时关联表（由 DataInitializer 维护）
```

---

## 七、侧边栏渲染机制

1. 用户登录后，前端调用 `/api/menu/current` 获取当前用户有权限的菜单列表
2. `MenuService` 查询用户角色 → 关联菜单 → 返回菜单树
3. 前端 `sidebar.js` 遍历菜单树，生成分组 + 菜单项 HTML
4. 无权限的菜单项不返回，前端侧边栏自然不会显示

> 💡 **安全提示**：侧边栏隐藏只是 UI 层面，用户直接访问 URL 仍可能进入页面。后端通过 `@RequirePermission` 注解对接口进行二次校验，是真正的安全保障。

---

## 八、修改菜单配置

修改 `menu-config.json` 后，需要**重启后端**才能生效（菜单数据在启动时加载到内存并缓存到 Redis）。

新增菜单项后，还需要：
1. 在 `permission-config.json` 中添加对应的按钮权限（如果需要）
2. 在 `role-config.json` 中给需要的角色添加该菜单的关联
