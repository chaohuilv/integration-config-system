# 权限配置指南

> 本文档详细介绍 `permission-config.json` 的配置结构、全部权限码以及如何新增自定义权限。

---

## 一、文件位置与结构

**路径**：`src/main/resources/config/permission-config.json`

**顶层结构**：

```json
{
  "permissions": [ /* 权限项数组 */ ]
}
```

---

## 二、字段说明

每个权限项的字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | string | ✅ | 权限码，唯一标识，格式 `{模块}:{操作}` |
| `name` | string | ✅ | 显示名称，如"新建接口" |
| `description` | string | ❌ | 详细描述，说明该权限控制的具体操作 |
| `module` | string | ✅ | 所属模块，用于分组展示 |
| `sortOrder` | number | ❌ | 排序序号（越小越靠前） |

---

## 三、权限码命名规范

格式：`{模块}:{操作}`

- **模块**：与功能模块对应，如 `api`、`env`、`user`、`role` 等
- **操作**：常用操作 `view` / `add` / `edit` / `delete`，以及模块特定操作如 `detail` / `invoke` / `export` / `doc`

---

## 五、前端使用方式

### 方式一：声明式（推荐）

在 HTML 元素上添加 `data-permission` 属性，无权限时自动隐藏：

```html
<button data-permission="api:add" onclick="showForm()">新建接口</button>
<button data-permission="api:delete" onclick="deleteApi(id)">删除</button>
```

> `permission.js` 在页面加载时自动扫描并处理。

### 方式二：编程式

在 JavaScript 中手动判断：

```javascript
if (Permission.check('api:delete')) {
    // 有权限，启用删除按钮
}
```

---

## 六、后端使用方式

在 Controller 方法上添加 `@RequirePermission` 注解：

```java
@RequirePermission("api:delete")
@DeleteMapping("/{id}")
public Result<Void> deleteApi(@PathVariable Long id) {
    apiConfigService.delete(id);
    return Result.success();
}
```

无权限时后端返回 HTTP 403，并附带错误信息。

## 七、如何新增权限码

### 步骤 1：在 permission-config.json 添加定义

```json
{
  "code": "api:import",
  "name": "导入接口",
  "description": "从 Curl 或 OpenAPI 导入接口配置",
  "module": "api",
  "sortOrder": 11
}
```

### 步骤 2：在 Controller 方法上添加注解

```java
@RequirePermission("api:import")
@PostMapping("/import")
public Result<Void> importApi(@RequestBody ImportDTO dto) {
    // ...
}
```

### 步骤 3：在前端页面使用权限属性

```html
<button data-permission="api:import" onclick="showImportDialog()">导入接口</button>
```

### 步骤 4：给角色分配该权限

通过角色编辑页面，将新权限分配给需要的角色（ADMIN 角色自动拥有所有权限，无需单独配置）。

---

## 八、权限与菜单的关系

权限控制的是**操作**，菜单控制的是**页面**。两者独立配置：

```
权限码 "api:import" 控制"导入按钮"是否显示
菜单项  "import"     控制"导入页面"是否在侧边栏显示
```

一个功能通常需要同时配置菜单和权限。例如"接口导入"功能：
- 菜单项：`import`（Curl导入）→ 控制页面是否可见
- 权限码：`api:import` → 控制导入按钮是否可用
