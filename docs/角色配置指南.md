# 角色配置指南

> 本文档详细介绍 `role-config.json` 的配置结构、三种内置角色以及如何创建自定义角色。

---

## 一、文件位置与结构

**路径**：`src/main/resources/config/role-config.json`

**顶层结构**：

```json
{
  "roles": [ /* 角色数组 */ ]
}
```

---

## 二、字段说明

每个角色的字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | string | ✅ | 角色唯一标识，唯一且不可更改 |
| `name` | string | ✅ | 显示名称 |
| `description` | string | ❌ | 描述说明 |
| `isSystem` | boolean | ✅ | 是否系统内置角色（`true` 时禁止删除） |
| `sortOrder` | number | ❌ | 排序序号 |
| `menus` | array | ❌ | 该角色可见的菜单 code 数组 |
| `permissions` | array | ❌ | 该角色拥有的权限码数组 |

> **重要**：`isSystem: true` 的角色拥有所有菜单和所有权限（动态从数据库加载），无需在 JSON 中配置 `menus` 和 `permissions`。

---

## 三、三种内置角色

### 3.1 系统管理员（ADMIN）

```json
{
  "code": "ADMIN",
  "name": "系统管理员",
  "description": "拥有系统全部权限，可管理所有功能模块",
  "isSystem": true,
  "sortOrder": 1
}
```

**特点**：
- `isSystem: true`，禁止删除
- 不需要配置 `menus` 和 `permissions`，启动时 `DataInitializer` 自动赋予所有菜单和所有权限
- 通常分配给技术负责人或运维人员

### 3.2 开发者（DEVELOPER）

```json
{
  "code": "DEVELOPER",
  "name": "开发者",
  "description": "可管理接口配置、进行调试操作、查看日志",
  "isSystem": true,
  "sortOrder": 2,
  "menus": ["list", "import", "debug", "logs", "environments"],
  "permissions": [
    "api:view", "api:detail", "api:add", "api:edit", "api:delete", "api:invoke",
    "api:version", "api:deprecate",
    "env:view", "env:detail", "env:add", "env:edit", "env:delete",
    "log:view", "invoke-log:detail"
  ]
}
```

**特点**：
- `isSystem: true`，禁止删除
- 可使用接口配置、Curl导入、接口调试、调用日志、环境配置的全部功能
- 无法管理用户、角色、系统配置
- 通常分配给后端/前端开发工程师

### 3.3 只读用户（READONLY）

```json
{
  "code": "READONLY",
  "name": "只读用户",
  "description": "只能查看接口配置和日志，无增删改权限",
  "isSystem": true,
  "sortOrder": 99,
  "menus": ["debug", "logs"],
  "permissions": ["api:invoke", "log:view", "invoke-log:detail"]
}
```

**特点**：
- `isSystem: true`，禁止删除
- 只能看到接口调试和调用日志菜单
- 只能调用接口（无增删改），无法访问其他任何功能
- 适合测试人员、甲方对接人员

## 四、自定义角色

### 4.1 在 role-config.json 中添加

```json
{
  "code": "VIEWER",
  "name": "接口查看者",
  "description": "只读查看接口配置，不能进行任何修改操作",
  "isSystem": false,
  "sortOrder": 10,
  "menus": ["list"],
  "permissions": ["api:view", "api:detail"]
}
```

### 4.2 配置说明

| 配置项 | 说明 |
|--------|------|
| `code` | 建议大写下划线命名，唯一即可 |
| `isSystem` | 必须为 `false`，`true` 仅限内置角色 |
| `menus` | 指定该角色在侧边栏能看到的页面 |
| `permissions` | 指定该角色在页面上能执行的操作 |

### 4.3 典型角色配置示例

**接口查看者（只读查看）**：
```json
{
  "code": "VIEWER",
  "name": "接口查看者",
  "menus": ["list", "logs"],
  "permissions": ["api:view", "api:detail", "log:view", "invoke-log:detail"]
}
```

**测试人员（可调用接口）**：
```json
{
  "code": "TESTER",
  "name": "测试人员",
  "menus": ["list", "debug", "logs"],
  "permissions": ["api:view", "api:detail", "api:invoke", "log:view", "invoke-log:detail"]
}
```

**运维人员（管理环境）**：
```json
{
  "code": "OPS",
  "name": "运维人员",
  "menus": ["list", "environments", "logs", "alert-rules", "alert-records"],
  "permissions": [
    "api:view", "api:detail", "api:invoke",
    "env:view", "env:detail", "env:add", "env:edit", "env:delete",
    "log:view", "invoke-log:detail",
    "alert:view", "alert:add", "alert:edit", "alert:delete"
  ]
}
```

## 五、角色与菜单/权限的关系

```
┌─────────────────────────────────────────────────────┐
│                    role-config.json                  │
│                                                     │
│  角色 = menus（菜单列表）+ permissions（权限列表）   │
│                                                     │
└──────────┬──────────────────────────┬──────────────┘
           │                          │
           ▼                          ▼
┌──────────────────┐        ┌──────────────────────────┐
│  menu-config.json │        │  permission-config.json  │
│   菜单结构定义     │        │   权限码定义              │
└──────────────────┘        └──────────────────────────┘
           │                          │
           ▼                          ▼
      侧边栏页面可见性             按钮操作可见性
```

**一个用户可能有多个角色**，最终可见范围是各角色权限的**并集**。

---

## 六、DataInitializer 初始化逻辑

系统启动时，`DataInitializer` 读取 `role-config.json`，执行以下操作：

1. **创建/更新角色**：如果角色不存在则创建，已存在则更新基本信息
2. **ADMIN 角色特殊处理**：无论是否已存在，重新赋予所有菜单和所有权限
3. **其他内置角色**：如果 `menus`/`permissions` 有变化，更新数据库
4. **跳过 isSystem: false 的自定义角色**的菜单/权限更新（由管理员在页面上手动配置）

---

## 七、修改角色配置后生效

修改 `role-config.json` 后重启后端即可生效。

新增自定义角色后，需要在**角色管理页面**上手动给角色分配菜单和权限（因为 `DataInitializer` 对 `isSystem: false` 的角色不自动同步菜单/权限）。
