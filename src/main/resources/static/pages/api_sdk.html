1. 认证机制

🔐 SDK 认证流程

  1. 调用 client.login(userCode, password) 登录，系统返回 access_token
  2. SDK 自动将 Authorization: Bearer <token> 添加到所有后续请求头中
  3. Token 有效期约 7 天，过期后调用接口会抛出 IntegrationException(认证失败)
  4. 调用 client.logout() 可主动注销 Token

2. Java SDK

2.1 快速开始

// 初始化客户端 IntegrationClient client = new IntegrationClient("http://localhost:8080"); // ① 登录认证（必须的第一步） AuthResult auth = client.login("admin", "123456"); System.out.println("欢迎: " + auth.getDisplayName()); // ② 调用接口（无需手动传 Token，SDK 自动携带） String result = client.invoke("user-get", Map.of("id", 1)); // POST 请求 String result = client.invoke("user-create", Map.of("deptId", 10), // URL 参数 "{\"name\":\"张三\",\"age\":25}" // 请求体 ); // 带自定义请求头（可覆盖默认认证头） Map<String, String> headers = new HashMap<>(); headers.put("X-Request-Id", "12345"); String result = client.invoke("user-create", Map.of(), "{\"name\":\"张三\"}", headers ); // ③ 退出登录 client.logout();

2.2 完整示例

import com.integration.sdk.IntegrationClient; import com.integration.sdk.IntegrationClient.AuthResult; import java.util.Map; public class Example { public static void main(String[] args) { IntegrationClient client = new IntegrationClient("http://localhost:8080"); try { // 登录 AuthResult auth = client.login("admin", "123456"); System.out.println("登录成功: " + auth.getDisplayName() + " (" + auth.getUserCode() + ")"); // 查询用户 String userJson = client.invoke("user-get", Map.of("id", 1001)); System.out.println("用户信息: " + userJson); // 创建用户 String newUserJson = client.invoke("user-create", Map.of("deptId", 10), "{\"username\":\"zhangsan\",\"email\":\"zhangsan@example.com\"}"); System.out.println("创建结果: " + newUserJson); // 异步调用 CompletableFuture<String> future = client.invokeAsync( "user-list", null, null, null); String asyncResult = future.get(); System.out.println("异步结果: " + asyncResult); // 退出 client.logout(); } catch (IntegrationException e) { System.err.println("调用失败: " + e.getMessage()); } } }

6. 请求头与认证

✅ 自动认证：调用 login() 后，SDK 会自动在所有 invoke() 请求中添加 Authorization: Bearer <token>，无需手动处理。

📋 请求头优先级规则：

SDK 自动添加Authorization: Bearer <token>（登录后自动携带）
默认请求头Content-Type: application/json / Accept: application/json
用户自定义 headers最高优先级，可覆盖默认/认证请求头
// 示例：调试时用自定义 Token 覆盖认证头
Map<String, String> headers = new HashMap<>();
headers.put("Authorization", "Bearer debug-token-xxx");
client.invoke("user-get", Map.of("id", 1), null, headers);
// 此时使用 debug-token-xxx，而非 login() 获取的 Token

7. 后端接口认证配置（可选）

如果目标接口还需要额外的认证（如第三方 API Key、Bearer Token 等），可以在接口配置页面中设置。 这与 SDK 自身登录认证是独立的两个层次。

常见的 Header 认证方式

🔹 API Key 认证

认证类型: API_KEY | 认证位置: HEADER | 认证参数名: X-API-Key
认证值: sk-xxxxxxxxxxxxxx
→ 请求头: X-API-Key: sk-xxxxxxxxxxxxxx

🔹 Bearer Token 认证（OAuth 2.0）

认证类型: BEARER | 认证位置: HEADER | 认证参数名: Authorization
认证值: Bearer eyJhbGciOiJIUzI1NiIs...
→ 请求头: Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

🔹 Basic Auth 认证

认证类型: BASIC | 认证位置: HEADER | 认证参数名: Authorization
认证值: Basic base64(username:password)
→ 请求头: Authorization: Basic dXNlcjpwYXNz

🔹 动态 Token 认证（自动刷新）

认证类型: DYNAMIC
Token接口编码: auth-token-get（系统中已配置的获取Token接口）
Token提取路径: $.data.accessToken（从Token接口响应中提取）
Token参数位置: HEADER | Token请求头参数名: Authorization | Token前缀: Bearer
Token缓存时间: 3600 秒（1小时）
→ 系统自动调用 auth-token-get → 提取 Token → 缓存 → 附加到请求头

🔄 场景编排 SDK

通过 SDK 编排多个接口调用，支持步骤依赖、缓存和失败策略

1. 概述

场景编排 SDK（ScenarioClient）复用 IntegrationClient 的登录状态，用于执行已配置的场景。 场景由多个步骤组成，每个步骤调用一个接口配置，支持步骤间参数传递、缓存和失败策略。
推荐使用场景编码（scenarioCode）调用，与业务解耦，不受数据库主键变化影响。

🔑 核心概念： 场景（Scenario）= 一组有序步骤（Step），每个步骤调用一个接口配置（ApiConfig）。 步骤间通过 ${step.步骤编码.字段名} 引用上一步输出，支持缓存命中跳过、失败策略（STOP / CONTINUE）。

2. Java 场景 SDK

2.1 快速开始

import com.integration.sdk.IntegrationClient; import com.integration.sdk.ScenarioClient; import com.integration.sdk.ScenarioClient.ScenarioResult; // 1️⃣ 先登录 IntegrationClient IntegrationClient client = new IntegrationClient("http://localhost:8080"); client.login("admin", "123456"); // 2️⃣ 用已登录的 client 创建 ScenarioClient ScenarioClient scenario = new ScenarioClient(client); // 3️⃣ 按场景编码执行（推荐） ScenarioResult result = scenario.execute("order-sync", Map.of("orderId", "20260425001")); System.out.println("执行状态: " + result.getStatus()); // SUCCESS / FAILED / PARTIAL System.out.println("执行耗时: " + result.getCostTimeMs() + "ms"); System.out.println("执行记录ID: " + result.getExecutionId()); // 异步执行（立即返回执行记录ID，后台继续执行） ScenarioResult asyncResult = scenario.execute("order-sync", Map.of("orderId", "20260425001"), true, // asyncExec = true "API" // triggerSource );

2.2 完整示例

import com.integration.sdk.IntegrationClient; import com.integration.sdk.ScenarioClient; import com.integration.sdk.ScenarioClient.ScenarioResult; import java.util.Map; public class ScenarioExample { public static void main(String[] args) { IntegrationClient client = new IntegrationClient("http://localhost:8080"); try { client.login("admin", "123456"); ScenarioClient scenario = new ScenarioClient(client); // 查询所有启用的场景 String activeList = scenario.getActiveScenarios(); System.out.println("启用场景: " + activeList); // 按编码执行（推荐） ScenarioResult result = scenario.execute("order-flow", Map.of("orderId", 12345, "userId", "U001")); if (result.getSuccess()) { System.out.println("✅ 执行成功"); System.out.println("场景: " + result.getScenarioName()); System.out.println("耗时: " + result.getCostTimeMs() + "ms"); } else { System.err.println("❌ 执行失败: " + result.getErrorMessage()); } // 按 ID 执行（兼容旧方式） ScenarioResult result2 = scenario.executeById(1L, Map.of()); // 查询场景详情 String detail = scenario.getDetail(1L); // 查询步骤列表 String steps = scenario.getSteps(1L); // 分页查询执行记录 String executions = scenario.listExecutions( null, "order-flow", "SUCCESS", 0, 20); // 查询执行记录详情 String execDetail = scenario.getExecution(100L); // 查询执行步骤日志 String execSteps = scenario.getExecutionSteps(100L); // 查询分组名称 String groups = scenario.getGroupNames(); // 分页搜索场景 String list = scenario.listScenarios( "订单组", null, "order", 0, 10); } catch (Exception e) { System.err.println("错误: " + e.getMessage()); } finally { client.logout(); } } }

2.3 ScenarioResult 执行结果

ScenarioResult result = scenario.execute("order-sync", params); result.getSuccess(); // Boolean — 是否全部成功 result.getExecutionId(); // Long — 执行记录ID result.getScenarioCode(); // String — 场景编码 result.getScenarioName(); // String — 场景名称 result.getStatus(); // String — 执行状态 // SUCCESS — 全部步骤成功 // FAILED — 有步骤失败且策略为 STOP // PARTIAL — 有步骤失败但策略为 CONTINUE // RUNNING — 异步执行中 result.getCostTimeMs(); // Long — 耗时（毫秒） result.getErrorMessage(); // String — 失败原因 result.getRawResponse(); // String — 原始响应JSON

6. 执行状态说明

状态说明
SUCCESS所有步骤执行成功
FAILED有步骤失败，且场景失败策略为 STOP（停止执行）
PARTIAL有步骤失败，但场景失败策略为 CONTINUE（继续执行）

7. 执行参数与步骤间引用

💡 步骤间参数传递： 通过 ${step.步骤编码.字段路径} 引用上一步输出。 例如步骤2引用步骤1的orderId：${step.query-order.orderId}。
也支持输入参数引用：${input.orderId}，即 execute() 传入的 params。

📌 参数引用语法：

${input.xxx}引用 execute() 传入的输入参数
${step.步骤编码.字段路径}引用上一步输出结果的字段
${token.接口编码}引用动态 Token 缓存值
${cache.缓存键}引用场景缓存数据
static:固定值静态值，不进行变量替换

8. 场景缓存

当步骤启用缓存（enableCache = true）且设置了缓存秒数（cacheSeconds > 0）时， 该步骤首次执行后会将结果缓存；后续执行时若缓存未过期则直接返回缓存结果，跳过实际 API 调用。

# Python 示例：启用缓存的场景，相同参数在缓存期内不会重复调用 result1 = scenario.execute("order-flow", params={"orderId": "A001"}) # 实际调用 API result2 = scenario.execute("order-flow", params={"orderId": "A001"}) # 缓存命中，跳过 API 调用 # Java 示例 ScenarioResult r1 = scenario.execute("order-flow", Map.of("orderId", "A001")); ScenarioResult r2 = scenario.execute("order-flow", Map.of("orderId", "A001")); // 缓存命中
