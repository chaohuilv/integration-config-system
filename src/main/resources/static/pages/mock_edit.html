🎯 匹配规则 （可选，不配置则仅匹配路径+方法）

💡 匹配规则填写说明 ▼
匹配来源 运算符 field（字段名） value（期望值）
请求头
HTTP Header		 等于 Header 名称，如 Authorization 期望的 Header 值，如 Bearer xxx
包含 Header 名称，如 Content-Type 值需包含的字符串，如 json
正则 Header 名称，如 User-Agent 正则表达式，如 Mozilla/5.*Windows
存在 Header 名称，如 X-Request-Id （留空即可）
查询参数
URL ?a=1&b=2		 等于 参数名，如 status 期望值，如 active
包含 参数名，如 keyword 值需包含的字符串，如 user
正则 参数名，如 phone 正则表达式，如 ^1[3-9]\d{9}$
存在 参数名，如 page （留空即可）
请求体
POST/PUT JSON		 等于 JSON 字段名，如 code 或 user.id 期望值，如 11
包含 JSON 字段名，如 message 值需包含的字符串，如 error
正则 JSON 字段名，如 email 正则表达式，如 ^[\w.-]+@[\w.-]+$
存在 JSON 字段名，如 data （留空即可）
JSON Path JSON Path 表达式，如 $.code 期望值，如 11
📌 填写示例：
场景：请求体 {"code":"11","name":"xxx"} 时返回 mock 响应
配置：匹配来源 = 请求体，运算符 = JSON Path
    field 填 $.code（从请求体中取 code 字段）
    value 填 11（期望 $.code 的值等于 11）
⚠ 注意：JSON Path 运算符仅适用于「请求体」，不能用于请求头或查询参数
